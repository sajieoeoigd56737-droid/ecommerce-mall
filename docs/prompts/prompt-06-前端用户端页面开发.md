# 第 6 部分：前端用户端页面开发提示词

```text
请继续完成 ecommerce-mall/frontend 的用户端页面开发。

用户端页面包括：

1. Login.vue 用户登录页
2. Register.vue 用户注册页
3. Home.vue 商城首页
4. ProductDetail.vue 商品详情页
5. Cart.vue 购物车 / 结算页
6. Orders.vue 我的订单页

要求：

1. 使用 Vue 3。
2. 使用 Element Plus。
3. 使用 Axios API 封装，不要直接在页面里写重复 axios。
4. 用户信息保存到 localStorage。
5. 不使用 Pinia。
6. 所有操作成功或失败都要有 ElMessage 提示。
7. 页面风格统一，温暖、简洁、现代。
8. 不要使用 mock 数据，必须调用真实后端接口。
9. 如果用户未登录，访问购物车、订单、加入购物车时要提示先登录并跳转 /login。

==================================================
整体 UI 风格
==================================================

请使用温暖、简洁、现代的电商系统风格。

配色：

- 背景：奶油白、浅米色、暖灰色
- 主色：柔和橙色 / 陶土橙
- 标题文字：深棕色或深灰色
- 普通文字：灰棕色
- 卡片：白色或浅米色
- 按钮：橙色主按钮，浅色描边次按钮

视觉：

- 圆角卡片
- 柔和阴影
- 留白充足
- 不要暗黑风
- 不要复杂动效
- 页面要像一个作业级但精致的电商系统
- 风格参考：温暖奶油色背景 + 橙色按钮 + 极简卡片式布局

==================================================
一、Login.vue 用户登录页
==================================================

功能：

- 用户输入 username 和 password
- 调用 user.js 中的 login(data)
- 登录成功后保存 userInfo 到 localStorage
- localStorage key 使用 userInfo
- 登录成功后跳转 /home
- 提供注册链接，点击跳转 /register

页面内容：

左侧品牌区域：
- logo
- 标题：简易商城系统
- 副标题：轻松购物，快乐生活
- 三个卖点：
  - 精选好物，品质保证
  - 安全支付，放心购买
  - 快速配送，售后无忧

右侧登录卡片：
- 标题：用户登录
- 用户名输入框
- 密码输入框
- 登录按钮
- 立即注册链接

校验：

- 用户名不能为空
- 密码不能为空

==================================================
二、Register.vue 用户注册页
==================================================

功能：

- 输入 username、password、nickname
- 调用 user.js 中的 register(data)
- 注册成功后跳转 /login
- 提供返回登录按钮

校验：

- 用户名不能为空
- 密码不能为空
- 昵称可以为空

页面风格和 Login.vue 保持一致。

==================================================
三、Home.vue 商城首页
==================================================

功能：

- 页面加载时调用 product.js 中的 getProductList()
- 展示商品列表
- 点击商品卡片或"查看详情"按钮跳转 /product/:id

页面内容：

顶部导航栏：
- 简易商城系统 logo
- 首页
- 购物车
- 我的订单
- 后台入口
- 搜索框
- 购物车图标
- 用户图标

导航跳转：
- 首页 → /home
- 购物车 → /cart
- 我的订单 → /orders
- 后台入口 → /admin/login

Banner 区域：
- 文案：品质生活 · 精选好物
- 副文案：全场满199减20，限时优惠
- 按钮：立即选购

商品区域：
- 标题：精选商品
- 商品卡片展示：
  - 商品图片
  - 商品名称
  - 商品分类
  - 商品价格
  - 商品库存
  - 查看详情按钮

==================================================
四、ProductDetail.vue 商品详情页
==================================================

功能：

- 根据路由 id 调用 product.js 中的 getProductDetail(id)
- 展示商品详情
- 输入购买数量
- 点击加入购物车调用 cart.js 中的 addCart(data)
- 未登录时提示请先登录并跳转 /login

页面内容：

- 顶部导航栏，风格与 Home.vue 一致
- 面包屑：首页 > 分类 > 商品名称
- 左侧商品图片
- 右侧商品信息：
  - 商品名称
  - 分类
  - 价格
  - 库存
  - 商品描述
  - 数量选择器
  - 加入购物车按钮
  - 立即购买按钮

说明：

- 立即购买按钮 MVP 可以只做展示，点击提示"功能开发中"。
- 数量不能小于 1。
- 数量不能超过库存。

==================================================
五、Cart.vue 购物车 / 结算页
==================================================

功能：

- 从 localStorage 获取 userInfo.id
- 调用 cart.js 中的 getCartList(userId)
- 展示购物车列表
- 支持修改数量
- 支持删除商品
- 自动计算总金额
- 填写收货信息
- 点击提交订单调用 order.js 中的 createOrder(data)
- 成功后跳转 /orders

页面内容：

顶部导航栏：
- 和 Home.vue 一致

左侧购物车商品表格：
- 商品
- 单价
- 数量
- 小计
- 操作

底部：
- 已选商品数量
- 合计金额

右侧收货信息卡片：
- 收货人 receiverName
- 手机号 receiverPhone
- 收货地址 receiverAddress
- 总金额
- 提交订单按钮

校验：

- 未登录，跳转 /login
- 购物车为空时显示空状态
- 收货人不能为空
- 手机号不能为空
- 收货地址不能为空
- 提交订单成功后清空页面购物车状态，并跳转 /orders

==================================================
六、Orders.vue 我的订单页
==================================================

功能：

- 从 localStorage 获取 userInfo.id
- 调用 order.js 中的 getOrderList(userId)
- 展示当前用户订单列表
- 如果订单状态为 0，显示"模拟支付"按钮
- 点击模拟支付调用 order.js 中的 payOrder(orderId)
- 支付成功后刷新订单列表

页面内容：

顶部导航栏：
- 和 Home.vue 一致

标题：
- 我的订单
- 查看和管理您的所有订单

状态 tab：
- 全部订单
- 待支付
- 已支付
- 已发货
- 已完成
- 已取消

订单卡片：
- 订单编号
- 下单时间
- 商品缩略图，如果后端返回 items，则展示 items 图片；如果没有图片，可以只展示商品名称
- 收货人
- 手机号
- 收货地址
- 订单金额
- 状态标签
- 模拟支付按钮
- 查看详情按钮

订单状态映射：

0 → 待支付
1 → 已支付
2 → 已发货
3 → 已完成
4 → 已取消

要求：

1. 支持 tab 筛选订单状态。
2. 待支付订单显示"模拟支付"按钮。
3. 非待支付订单不显示"模拟支付"按钮。
4. 查看详情按钮 MVP 可以只提示"功能开发中"。

==================================================
公共要求
==================================================

1. 用户端所有页面风格统一。
2. 顶部导航栏可以每个页面写一份，MVP 不强制抽组件。
3. 如果愿意，也可以抽成 components/AppHeader.vue。
4. 页面要能正常调用后端接口。
5. 字段名必须和后端返回一致。
6. 不要引入复杂状态管理。
```
