# 第 7 部分：前端管理后台页面开发提示词

```text
请继续完成 ecommerce-mall/frontend 的管理员后台页面。

管理员页面包括：

1. AdminLogin.vue 管理员登录页
2. AdminProduct.vue 商品管理页
3. AdminOrder.vue 订单管理页

要求：

1. 使用 Vue 3。
2. 使用 Element Plus。
3. 使用已有 api/admin.js 中封装的接口。
4. 管理员登录信息保存到 localStorage。
5. localStorage key 使用 adminInfo。
6. 不使用 Pinia。
7. 不使用 mock 数据。
8. 所有数据必须来自后端接口。
9. 操作成功或失败必须使用 ElMessage 提示。
10. 页面风格和用户端保持同一套暖色系视觉。

==================================================
整体后台 UI 风格
==================================================

后台页面风格：

- 左侧浅米色侧边栏
- 右侧白色内容卡片
- 主色使用柔和橙色
- 按钮、当前菜单高亮使用橙色
- 表格使用 Element Plus
- 弹窗使用 Element Plus Dialog
- 表单使用 Element Plus Form
- 圆角卡片
- 柔和阴影
- 不要暗黑风
- 不要复杂动效

==================================================
一、AdminLogin.vue 管理员登录页
==================================================

功能：

- 管理员输入 username 和 password
- 调用 admin.js 中的 adminLogin(data)
- 登录成功后保存 adminInfo 到 localStorage
- 跳转 /admin/products

页面内容：

- 标题：简易商城后台
- 副标题：管理员登录
- 用户名输入框
- 密码输入框
- 登录按钮
- 返回商城首页链接

校验：

- 用户名不能为空
- 密码不能为空

默认账号提示可以展示：

管理员账号：admin
密码：123456

==================================================
二、后台公共布局
==================================================

AdminProduct.vue 和 AdminOrder.vue 需要有统一后台布局。

左侧侧边栏：

标题：
- 简易商城后台

菜单：
- 首页概览
- 商品管理
- 订单管理
- 用户管理
- 分类管理
- 数据统计
- 系统设置
- 退出登录

菜单功能：

- 商品管理 → /admin/products
- 订单管理 → /admin/orders
- 退出登录 → 清除 adminInfo，并跳转 /admin/login
- 其他菜单 MVP 可以点击后提示"功能开发中"

右上角：

- 管理员头像或图标
- 管理员名称
- 下拉箭头，可不实现真实下拉

登录校验：

- 如果没有 adminInfo，访问 /admin/products 或 /admin/orders 时提示请先登录，并跳转 /admin/login

==================================================
三、AdminProduct.vue 商品管理页
==================================================

功能：

- 调用 admin.js 中的 getAdminProductList()
- 展示商品列表
- 支持新增商品
- 支持编辑商品
- 支持删除商品
- 使用 Element Plus Table
- 使用 Element Plus Dialog
- 使用 Element Plus Form

页面内容：

主体标题：
- 商品管理

工具栏：
- 搜索框，placeholder：搜索商品名称、分类...
- 新增商品按钮

商品表格字段：

- 商品ID
- 商品图
- 商品名称
- 分类
- 价格
- 库存
- 状态
- 操作

状态显示：

- status = 1 显示"上架"
- status = 0 显示"下架"

操作按钮：

- 编辑
- 删除

新增商品表单字段：

- name 商品名称
- category 商品分类
- price 商品价格
- stock 商品库存
- imageUrl 商品图片地址
- description 商品描述
- status 商品状态

编辑商品：

- 点击编辑按钮后打开弹窗
- 表单回显当前商品数据
- 修改后调用 updateProduct(data)
- 成功后关闭弹窗并刷新列表

删除商品：

- 点击删除按钮后弹出确认框
- 确认后调用 deleteProduct(id)
- 成功后刷新列表

搜索功能：

- MVP 可以前端本地过滤
- 根据商品名称或分类过滤

表单校验：

- 商品名称不能为空
- 价格不能为空，且不能小于 0
- 库存不能为空，且不能小于 0

==================================================
四、AdminOrder.vue 订单管理页
==================================================

功能：

- 调用 admin.js 中的 getAdminOrderList()
- 展示所有订单
- 如果订单状态为 1，显示"发货"按钮
- 点击发货调用 admin.js 中的 shipOrder(orderId)
- 发货成功后刷新列表

页面内容：

主体标题：
- 订单管理

订单表格字段：

- 订单编号
- 用户 ID
- 订单金额
- 收货人
- 手机号
- 收货地址
- 订单状态
- 创建时间
- 操作

订单状态映射：

0 → 待支付
1 → 已支付
2 → 已发货
3 → 已完成
4 → 已取消

状态标签颜色建议：

- 待支付：橙色
- 已支付：绿色
- 已发货：蓝色
- 已完成：灰色
- 已取消：红色或灰色

操作按钮：

- 发货
- 查看详情

规则：

- 只有 status = 1 的订单显示"发货"按钮
- 其他状态不显示发货按钮或禁用发货按钮
- 点击发货前弹出确认框
- 发货成功后刷新订单列表
- 查看详情按钮 MVP 可以提示"功能开发中"

==================================================
公共要求
==================================================

1. 后台页面必须和后端接口联调。
2. 不要使用 mock 数据。
3. 表格字段必须和后端返回字段一致。
4. 页面要简洁清晰，适合作业演示。
5. 商品管理和订单管理都要能真实操作数据库。
```
